# QuantRadar MVP — Design Document

Generated by /plan-ceo-review on 2026-03-20

## Vision

### 10x Check
The 10x version is a personal Bloomberg terminal: real-time streaming, AI market
commentary, multi-asset correlation analysis, sentiment from news/social, automated
portfolio rebalancing. That's ocean territory — deferred.

The achievable 3x version (what we're building): a complete quant workflow from
strategy design → backtest → live signals → decision tracking → performance review,
all in one Streamlit app with daily email digests.

## Key Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Stack | Streamlit Monolith | Fastest to working product, single language, free/cheap hosting |
| Storage | SQLite (WAL mode) | Zero-config, single file, WAL for concurrent cron+Streamlit access |
| KR Stocks | pykrx | OHLCV + fundamentals from KRX directly |
| KR Indices/FX | FinanceDataReader | KOSPI index, USD/KRW exchange rate (with yfinance fallback) |
| US Data | yfinance | Free, battle-tested, sufficient for daily data |
| Deploy | Local first → VPS later | Minimize complexity during MVP phase |
| Alerts | Email digest (not 카카오톡) | Simpler integration, equally effective for the use case |
| DB Config | Shared config/settings.py | Single source of truth for DB path, WAL mode, all cron+app imports |
| Strategy Schema | Single schema in strategy_engine.py | Templates are pre-filled instances of the same schema |
| Caching | st.cache_data(ttl=300) | Sub-second dashboard loads, 5-min cache refresh |
| Cron Monitoring | system_status table + health dot | Cron errors logged to SQLite, visible via health indicator |

## MVP Scope (PRD P0 + Accepted Expansions)

### Core (from PRD)
1. 종목 유니버스 설정 — KOSPI200 + S&P500 관심 종목 선택
2. 전략 빌더 — 조건 선택 방식 (PER, RSI, 이동평균, 거래량 등)
3. 백테스트 — 과거 3년 데이터 적용, 수익률 그래프
4. 실시간 신호 대시보드 — 매수/매도 조건 달성 종목 리스트
5. 신호 알림 — 이메일 기반

### Accepted Expansions
6. Pre-built strategy templates (5-7 battle-tested strategies)
7. Performance scoreboard (Sharpe ratio, max drawdown, win rate, vs benchmark)
8. Signal history & decision journal with outcome tracking
9. Dark mode + mobile-optimized Streamlit layout
10. Market context panel (KOSPI, S&P500, VIX, USD/KRW)
11. Daily morning email digest (replaces on-signal alerts)
12. Health indicator dot (data freshness + system status)

## Design Specifications (2026-03-20)

### Visual Identity
```
COLORS:
  Background:     #0E1117 (Streamlit dark default)
  Surface:        #262730
  Text Primary:   #FAFAFA
  Text Secondary: #A3A8B8
  Buy/Profit:     #00D26A (green)
  Sell/Loss:      #FF4B4B (red)
  Warning:        #FFA726 (amber)
  Neutral:        #636EFA (blue — charts)

TYPOGRAPHY: Source Sans Pro (Streamlit default)
  H1: 28px bold | H2: 22px semibold | Body: 16px | Caption: 14px | Metric: 32px bold

SPACING: 24px section gap, 16px card padding, 48px table row height (touch-friendly)
```

### Screen Wireframes

**Dashboard** (daily 5-second scan):
1st: Market context (4 metrics in a row: KOSPI, S&P500, VIX, USD/KRW)
2nd: Today's BUY/SELL signals (table — ticker, strategy, price, [Acted/Skipped] buttons)
3rd: Journal prompt ("Yesterday's signals: Did you act?")

**Strategy Builder:**
1st: Template buttons (Golden Cross, RSI Oversold, Value+Momentum, etc.)
2nd: Condition editor (dropdowns + value inputs, max 3 conditions)
3rd: Backtest results (scoreboard metrics via st.metric + equity curve line chart)

**Stock Detail** (drill-down from signal click):
1st: Candlestick chart with signal markers (Plotly financial preset)
2nd: Indicators (RSI, MA) + Fundamentals (PER, PBR) side by side
3rd: Signal history table with outcome column

**Settings:**
1st: Stock universe search + quick add buttons (KOSPI 50, S&P Top 30)
2nd: Email digest config (address, time, frequency, test button)

### First-Time Onboarding
3-step wizard on first launch:
1. "Pick some stocks" — pre-selected 10 (5 KR + 5 US blue chips), editable
2. "Choose a strategy" — template selection
3. "Run your first backtest" — instant results

### Interaction States
Every feature specifies: loading (skeleton/spinner), empty (warm message + action),
error (clear explanation), success (confirmation), partial (stale data warning).
See full state table in design review notes.

### Responsive Behavior
- Desktop: sidebar nav + wide main area, signals as table, market context 4-in-row
- Mobile (<768px): collapsed menu, signals as stacked cards, market context 2x2 grid
- All touch targets >= 48px, charts full-width on mobile

### Journal UX
One-click inline [Acted] / [Skipped] buttons on each signal row.
Optional notes field expands on click. No page navigation required.

### Anti-Slop Rules
- Signal list: TABLE, not cards (financial data is tabular)
- Scoreboard: st.metric with delta, not custom cards
- Charts: Plotly candlestick (financial), not generic line charts
- Colors: financial green/red, not SaaS pastels
- No hero sections, no generic card grids, no 3-column feature blocks
